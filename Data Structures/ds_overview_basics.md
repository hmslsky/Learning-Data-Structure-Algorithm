# 数据结构概述与基本操作

## 1. 数据结构的基本概念

数据结构是指相互之间存在一定关系的数据元素的集合。按照视点的不同，数据结构分为逻辑结构和存储结构：
- 从逻辑结构上看，数据结构分4类：集合、线性结构、树结构、图结构；
- 从存储结构看，常用存储结构有两种：顺序存储结构和链接存储结构。

## 2. 数据存储结构

数据的存储结构主要包括数据元素本身的存储以及数据元素之间关系表示。

### 2.1 数组（顺序存储）

数据元素的存储位置对应于一块连续的存储空间，数据元素之间的前驱和后续关系通过数据元素在存储器中的相对位置来反映。

**优点**
- 由于是紧凑连续存储，相对节约存储空间。
- 可以进行随机访问，通过索引可以快速找到对应元素。

**缺点**
- 由于是连续存储，导致内存空间必须一次性分配足够，如果数组要扩容，需要重新分配空间，然后把数据全部复制过去，时间复杂度为O(n)。
- 插入删除操作，需要移动后面的所有数据保持连续性，时间复杂度为O(n)。

### 2.2 链表（链式存储）

数据元素的存储对象的是不连续的存储空间，每个存储节点对应一个需要存储的数据元素。元素之间通过存储节点之间的链接关系反映出来。

**优点**
- 元素存储不连续，依靠指针插入删除操作，所以不存在数组扩容问题，插入删除元素时，修改指针就可以了，时间复杂度是O(1)。

**缺点**
- 由于存储空间不连续，相对而言会消耗更多的存储空间。
- 存储空间不连续，无法根据一个索引算出元素的地址，不能随机访问。

## 3. 基本操作

数据结构的基本操作无非就是遍历+访问，简而言之就是增删改查。

从遍历方式划分：
- 线性遍历，以for/while迭代为代表
- 非线性遍历，以递归为代表

### 3.1 线性遍历示例
```python
def linear_traverse(arr):
    for i in range(len(arr)):
        print(f"visit {arr[i]}")
```

### 3.2 非线性遍历示例
```python
# 二叉树遍历
def binary_tree_traverse(root):
    if root.left:
        binary_tree_traverse(root.left)
    if root.right:
        binary_tree_traverse(root.right)

# N叉树遍历
def ntree_traverse(root):
    print(f'Has visited {root.val}')
    if len(root.children) > 0:
        for child in root.children:
            ntree_traverse(child)
```

### 3.3 链表遍历
```python
# 迭代方式
def iterative_traverse(head):
    p = head
    while p is not None:
        print(f'visit {p.val}')
        p = p.next

# 递归方式
def recursive_traverse(head):
    print(f'visit {head.val}')
    recursive_traverse(head.next)
```

## 4. 抽象数据类型

数据类型（data type）是一组性质相同的数据元素的集合以及加在这个集合上的一组操作。例如：基本数值类型的加减乘除...

### 4.1 定义数据类型的作用
- 隐藏计算机硬件及其特性和差别，使硬件对于用户而言是透明的，即用户不用关心数据类型是怎么实现的而使用它；
- 使用户可以数据类型定义的操作，方便的实现问题的求解；

### 4.2 抽象数据类型（ADT）
抽象数据类型是一种数据模型，包括定义和实现两个方面，其中定义是独立于实现的。

- 抽象数据类型的定义取决于它的逻辑特性，而与计算机内部的实现无关，即无论内部结构怎么变化，只要它的逻辑特性不变，都不会影响到它的使用。
- 其内部的变化（抽象数据类型实现的变化）只是可能对外部在使用它解决问题时的效率上产生影响。

抽象数据类型可以使用一个三元组来表示：
```
ADT = (D, S, P)
```
其中：
- D 是数据对象
- S 是D上的关系集
- P 是加在D上的一组操作

ADT的基本定义格式：
```
ADT 抽象数据类型名 {
    数据对象：<数据对象的定义>
    数据关系：<数据关系的定义>
    基本操作：<基本操作的定义>
}
```
